既知の問題と教訓（上位5件）

1. Prisma enum import混在

@rakuda/database と @prisma/client が混在 → TSエラー316件

対策: ci.ymlにprisma generate追加済み

2. Coolify NODE_ENV問題

NODE_ENV=productionがビルド時に渡される → devDependenciesスキップ

対策: Dockerfileで NODE_ENV=development npm ci

3. マイグレーション実行漏れ

コードは新スキーマ参照、本番DBは旧スキーマ

対策: デプロイ手順にマイグレーション必須化

4. sourceType大文字小文字

MERCARI vs mercari → フィルタ失敗、クラッシュ

対策: 全レイヤーUPPERCASE統一済み

5. Turboキャッシュで古いコード残存

TURBO_FORCE=1 env変数は無効。古いPrismaクライアント（joomListing）が残存

対策: npx turbo run build --force（CLIフラグ）を使用

6. Joom API側でPAUSED検出

joom-status-sync定期ジョブがJoom API側のdisabled状態を検出→DB側をPAUSEDに更新。SBDC101の歴史的価格バグ(13M USD)が原因の可能性

対策: Worker再デプロイ後にenable APIジョブ処理で復旧予定
